# 배팅 게임

2~20개 팀이 2~20라운드 동안 `1~라운드 수`를 한 번씩 사용하며 벌이는 수업용 전략 게임입니다. 같은 라운드에 같은 숫자가 둘 이상 나오면 그 숫자는 모두 무효가 되고, 남은 숫자 중 가장 큰 수를 낸 팀이 그 수만큼 점수를 얻습니다.

프론트엔드는 정적 HTML/CSS/JavaScript로 실행되고, Google Apps Script가 구글 스프레드시트의 `시트1`을 데이터 저장소로 사용합니다. 별도 서버나 데이터베이스는 필요하지 않습니다.

## 파일 구성

- `index.html`: 관리자/참여자 화면
- `style.css`: 반응형 화면 디자인
- `app.js`: 화면 동작, JSONP API 통신, 로컬 저장
- `Code.gs`: Google Apps Script 백엔드 전체 코드
- `README.md`: 설치와 수업 운영 설명서

## 준비물

- 교사용 구글 계정
- 새 구글 스프레드시트 1개
- 최신 웹 브라우저
- 학생 팀별 스마트폰·태블릿·노트북 중 하나
- 관리자 화면을 보여 줄 빔프로젝터(권장)

## 1. 구글 스프레드시트 만들기

1. 새 구글 스프레드시트를 만듭니다.
2. 아래쪽 시트 탭의 이름을 `시트1`로 둡니다.
3. 다른 이름이라면 `시트1`로 바꿉니다.
4. 스프레드시트 주소를 복사해 둡니다.
5. 시트 안에는 표를 직접 만들지 않아도 됩니다. 게임 생성 시 `A1:H470` 영역이 자동으로 구성됩니다.

게임 생성은 `시트1`의 `A1:H470`을 초기화합니다. 같은 영역에 필요한 자료가 있다면 다른 시트나 파일로 옮겨 주세요.

## 2. Apps Script 코드 붙여넣기

1. 배팅 게임의 `관리자 모드`를 열고 `코드 전체 복사` 버튼을 누릅니다.
2. 스프레드시트 상단에서 `확장 프로그램 → Apps Script`를 엽니다.
3. 기본 편집기의 기존 코드를 모두 지우고 복사한 코드를 붙여넣습니다.
4. 프로젝트 이름을 알아보기 쉽게 `배팅 게임`으로 정합니다.
5. 저장 버튼을 누릅니다.

## 3. Apps Script를 웹앱으로 배포하기

1. Apps Script 편집기 오른쪽 위에서 `배포 → 새 배포`를 누릅니다.
2. 배포 유형에서 `웹 앱`을 선택합니다.
3. 실행 사용자는 `나`로 설정합니다.
4. 액세스 사용자는 학생들이 접속 가능한 범위로 설정합니다. 학교 계정 정책에 따라 `모든 사용자` 또는 조직 내 사용자를 선택할 수 있습니다.
5. `배포`를 누르고 처음 나타나는 권한 요청을 승인합니다.
6. 생성된 웹 앱 URL을 복사합니다. 정상 배포 URL은 보통 `/exec`로 끝납니다.

`Code.gs`를 나중에 수정했다면 `배포 → 배포 관리 → 수정 → 새 버전 → 배포` 순서로 새 버전을 반영해야 합니다. 단순 저장만으로 기존 `/exec` 배포가 갱신되지 않을 수 있습니다.

## 4. 프론트엔드에서 연결하기

1. `index.html`을 웹 서버 또는 GitHub Pages에서 엽니다.
2. `관리자 모드`를 선택합니다.
3. `Apps Script 웹앱 URL`에 위에서 복사한 `/exec` 주소를 붙여넣습니다.
4. `구글 스프레드시트 주소`에 준비한 스프레드시트 주소를 붙여넣습니다.
5. 팀 수와 라운드 수를 각각 `2~20` 사이에서 설정합니다.
6. `게임 생성`을 누르고 초기화 확인 창에서 확인합니다.
7. 설정한 팀 수만큼 중복 없는 4자리 참여 코드가 나타나는지 확인합니다.

웹앱 URL과 스프레드시트 주소는 현재 브라우저의 `localStorage`에 저장되므로 새로고침 후에도 다시 입력할 필요가 없습니다.

## 5. 관리자 모드 사용 순서

1. 게임을 생성합니다.
2. 화면에 나온 공통 QR을 학생들이 스캔하게 합니다.
3. 각 팀에는 자기 팀의 4자리 참여 코드만 개별 전달합니다.
4. 모든 팀이 접속할 준비가 되면 `게임 시작`을 누릅니다.
5. 게임판에서 각 팀의 제출 상태를 확인합니다.
6. 제출한 팀의 칸에는 숫자 대신 `제출 완료`가 표시됩니다.
7. 접속이 끊겼거나 온라인 제출이 어려운 팀은 현재 라운드의 `직접 입력` 칸을 눌러 남은 숫자 중 하나를 대신 제출합니다.
8. 개별 `제출 완료` 칸을 눌러 하나씩 공개하거나, 설정한 모든 팀이 제출한 후 `현재 라운드 전체 공개`를 누릅니다.
9. 모든 숫자가 공개되면 서버가 점수를 자동 계산합니다.
10. 결과를 확인한 뒤 `다음 라운드`를 누릅니다.
11. 설정한 마지막 라운드 결과가 계산되면 게임이 자동 종료되고 최종 순위가 표시됩니다.

관리자 화면은 5초마다 자동 동기화됩니다. 즉시 확인하려면 `동기화`를 누르세요.

팀 수·라운드 수 저장 구조가 바뀐 버전에서는 기존 게임을 불러오지 말고 `게임 생성`을 다시 눌러 시트 영역을 새 형식으로 초기화해야 합니다.

## 6. 참여자 모드 사용 순서

1. 관리자 화면의 공통 QR을 스캔합니다.
2. Apps Script URL과 스프레드시트 ID가 자동으로 입력됩니다.
3. 자기 팀의 4자리 참여 코드만 입력합니다.
4. `팀으로 접속하기`를 누릅니다.
5. 게임이 시작되면 남은 숫자 중 하나를 고르고 제출합니다.
6. 제출 전 확인 창에서 숫자를 다시 확인합니다. 제출 후에는 바꿀 수 없습니다.
7. 선생님이 다음 라운드로 이동하면 참여자 화면도 최대 5초 안에 갱신됩니다.

참여자 응답에는 자기 팀 정보와 자기 제출값만 포함됩니다. 다른 팀의 참여 코드나 제출 숫자는 참여자 화면에 전달하지 않습니다.

## 7. 점수 계산 규칙

- 팀은 매 라운드 `1~라운드 수` 중 하나를 제출합니다.
- 한 팀은 같은 숫자를 두 번 사용할 수 없습니다.
- 같은 라운드에서 둘 이상의 팀이 같은 숫자를 내면 그 숫자는 모두 무효입니다.
- 중복되지 않은 숫자 중 가장 큰 숫자를 낸 팀이 그 숫자만큼 점수를 얻습니다.
- 모든 숫자가 중복되었다면 그 라운드는 점수 획득 팀이 없습니다.

예: `1팀=6, 2팀=6, 3팀=5, 4팀=3, 5팀=2`이면 `6`은 중복으로 무효이고 `3팀`이 `5점`을 얻습니다.

## 8. 최종 순위 계산 규칙

순위는 반드시 Apps Script가 계산한 결과를 사용합니다.

1. 총점이 큰 팀이 앞섭니다.
2. 총점이 같으면 획득 점수들을 내림차순으로 비교합니다. `[5,1]`은 `[4,2]`보다 앞섭니다.
3. 점수 조합도 같으면 큰 점수를 더 앞 라운드에서 얻은 팀이 앞섭니다.
4. 위 조건까지 모두 같으면 같은 순위로 표시하며, 1위가 여러 팀이면 `공동 우승`으로 표시합니다.

## 9. 시트에 저장되는 영역

- `A1:H11`: 게임 설정
- `A12:H33`: 최대 20개 팀 정보와 참여 코드
- `A36:H437`: 최대 400개 제출 기록
- `A440:H461`: 최대 20개 라운드 결과

배팅 게임이 사용하는 영역 밖의 셀은 초기화하지 않습니다.

## 10. 실제 수업 진행 예시

1. 교사는 수업 전에 스프레드시트와 웹앱 배포를 준비합니다.
2. 교실에서 관리자 화면을 빔프로젝터에 띄우고 게임을 생성합니다.
3. 학생들을 설정한 수만큼 팀으로 나누고 자기 팀 코드만 전달합니다.
4. 규칙을 설명한 뒤 팀별로 30초~1분의 전략 회의 시간을 줍니다.
5. 각 팀이 숫자를 제출하면 제출 현황만 함께 확인합니다.
6. 모든 팀 제출 후 숫자를 하나씩 공개해 긴장감을 만들거나 전체 공개합니다.
7. 중복 숫자가 왜 무효가 되었는지, 어떤 전략이 효과적이었는지 짧게 이야기합니다.
8. 마지막 라운드 후 최종 순위와 동점 처리 과정을 함께 확인합니다.

## 11. 테스트용 시나리오

배포 후 실제 수업 전에 다음 순서로 점검하세요.

1. 관리자 모드에서 기본값인 5팀·6라운드 게임을 생성하고 5개 코드를 확인합니다.
2. 시크릿 창 또는 다른 브라우저 탭 5개에서 각 팀 코드로 접속합니다.
3. 1라운드에 차례로 `6, 6, 5, 3, 2`를 제출합니다.
4. 관리자 화면에서 제출 숫자가 바로 보이지 않고 `제출 완료`만 표시되는지 확인합니다.
5. 전체 공개 후 6이 무효 처리되고 3팀이 5점을 얻는지 확인합니다.
6. 이미 사용한 숫자가 해당 팀 화면에서 비활성화되는지 확인합니다.
7. 6라운드까지 진행하여 최종 순위가 표시되는지 확인합니다.
8. 별도로 20팀·20라운드 게임을 생성해 참여 코드, 게임판, 숫자 1~20이 모두 표시되는지 확인합니다.
9. 관리자 화면을 새로고침해도 상태가 유지되는지 확인합니다.

## 12. 자주 발생하는 오류와 해결 방법

### Apps Script에 연결할 수 없습니다

- 웹앱 URL이 `/exec`로 끝나는지 확인합니다.
- Apps Script의 웹앱 액세스 권한을 확인합니다.
- 학교 계정에서 외부 사용자의 접근을 막는 정책이 있는지 관리자에게 확인합니다.
- 코드를 수정했다면 새 버전으로 다시 배포합니다.

### 스프레드시트 또는 게임 ID를 찾을 수 없습니다

- 스프레드시트 주소를 주소창에서 다시 복사합니다.
- Apps Script를 해당 스프레드시트의 `확장 프로그램` 메뉴에서 만든 컨테이너 바인딩 프로젝트인지 확인합니다.
- 게임 ID 대신 전체 스프레드시트 주소를 사용하면 가장 확실합니다.

### 참여 코드가 존재하지 않습니다

- 숫자 4자리를 정확히 입력했는지 확인합니다.
- 게임 초기화 후에는 참여 코드가 새로 만들어지므로 최신 코드를 사용해야 합니다.

### 숫자를 제출할 수 없습니다

- 관리자가 `게임 시작`을 눌렀는지 확인합니다.
- 이번 라운드에 이미 제출했는지 확인합니다.
- 해당 숫자를 이전 라운드에서 이미 사용했는지 확인합니다.
- 참여자 기기에서 제출하기 어렵다면 관리자가 현재 라운드의 `직접 입력`으로 대신 제출할 수 있습니다.

### 전체 공개 또는 다음 라운드가 되지 않습니다

- 설정한 모든 팀이 제출해야 전체 공개할 수 있습니다.
- 모든 숫자가 공개되고 점수 계산이 끝나야 다음 라운드로 이동할 수 있습니다.
- `동기화`를 눌러 최신 상태를 다시 읽습니다.

### 상태가 갱신되지 않습니다

- 네트워크 연결을 확인합니다.
- 관리자 화면의 `동기화`를 누릅니다.
- 참여자 화면은 5초마다 자동 갱신됩니다. 필요하면 페이지를 새로고침해도 서버 상태는 유지됩니다.

## 통신 방식과 주의 사항

정적 페이지와 Apps Script 사이의 브라우저 CORS 제한을 피하기 위해 모든 프론트엔드 요청은 JSONP `GET` 방식으로 전송됩니다. Apps Script는 변경 요청마다 스크립트 잠금을 사용해 두 팀이 동시에 제출할 때 데이터가 겹치지 않도록 합니다.

이번 버전은 수업 편의를 위해 별도 로그인이나 관리자 비밀번호를 사용하지 않습니다. 참여 코드를 아는 사람은 해당 팀으로 접속할 수 있으므로 코드를 팀 밖에 공유하지 않도록 안내하세요.
